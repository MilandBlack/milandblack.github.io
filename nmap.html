Nmap: All commands may need sudo

To install nmap on linux use: sudo apt-get install nmap

To install nmap on Windows click here.

The subnet length is used when running a full network scan. This can be omitted if running a scan on a specific machine.

Vulners may not come with the installed version of nmap. Use: cd /usr/share/nmap/scripts/ to traverse to the scripts directory. Then use the command: git clone https://github.com/vulnersCom/nmap-vulners.git to install the script.

While still in the /usr/share/nmap/scripts directory use the following commands:

  • git clone https://github.com/scipag/vulscan.git
  • ln -s `pwd`/scipag_vulscan /usr/share/nmap/scripts/vulscan
  • cd vulscan/utilities/updater/
  • chmod +x updateFiles.sh
  • ./updateFiles.sh

Vulscan will take a long time if the database isn't specified so you may want to do them one at a time. If you want to do them altogether omit the --script-args section of the command.

The following vulnerability databases should be included in Vulscan.

  • exploitdb.csv
  • osvdb.csv
  • securitytracker.csv
  • openvas.csv
  • scipvuldb.csv
  • xforce.csv
  • securityfocus.csv
  • cve.csv

Scans

nmap [targetIP/subnetLength] - Initial scan to check ports.

nmap -sV --script vuln [targetIP/subnetLength]

nmap -sV --script vulners [targetIP]

nmap -sV --script vulscan --script-args vulscandb=[databaseName] [targetIP]

Prioritize!

These steps should set you on a good path. After these steps are implemented, be sure to complete as many injections as possible using the time alottments as a priority scale.
